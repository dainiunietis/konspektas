    Gulp'o command prompt / command line / terminal komandos

    npm komandos

    Gulp'o globalus instaliavimas

    Instaliuoja Gulp'ą globaliai — gulp command line utility — įvykdyti galima bet kuriame kataloge:

    > npm install --global gulp-cli

     

    Gulp'o instaliacija projekto kataloge

    1. Sukurti projekto katalogą (jeigu jo nėra)
    • arba su failų explorer'iu ar redaktoriuje, ar kaip kitaip,

    • arba per Command Prompt'ą Create a project directory and navigate into it:

      > npx mkdirp my-project
> cd my-project
    2. Sukurti failą package.json šakniniame projekto kataloge — initialize project directory

    Esant šakniniame projekto kataloge (project root):

    > npm init

    The npm init command creates a package.json file for your project which stores information about the project, like the dependencies used in the project (Gulp is an example of a dependency)

    2. Instaliuoti Gulp'ą projekto kataloge Install the gulp package in your devDependencies:

    > npm install --save-dev gulp

    This time, we're installing Gulp into project instead of installing it globally, which is why there are some differences in the command.

    You'll see that the sudo keyword isn't required because we're not installing Gulp globally, so -g is also not necessary. We've added --save-dev, which tells the computer to add gulp as a dev dependency in package.json.

     

    Gulp'o plugin'o instaliacija projekto kataloge

    > npm i --save-dev gulp-plugin-name

     

     

     

    Gulp'o komandos

    Įvykdo public Gulp'o task'ą:

    > gulp taskName

    Default'inis task'as

    Jeigu yra apibrėžtas default'inis task'as:

    
exports.default = series(kazkas1, kazkas2);

    Tada iškviečiant per CMD nereikia jo nurodyti:

    > gulp

     

     

    Išveda public Gulp'o task'us:

    > gulp --tasks
