    Yo Code - Extension Generator # New Color Theme

    How to write a Visual Studio Code color theme from scratch

    Kaip klonuoti aktyvią Color Theme'ą sukuriant VS Code'o Extension'ą su tuo Color Theme'u, t. y. pridėtų naują Color Theme'ą su nauju pavadinimu

    1. Jeigu nėra globaliai suinstaliuotų npm'o package'ų "Yeoman" ir "VS Code Extension generator" — reikia suinstaliuoti:

      Install Yeoman and the VS Code Extension generator from the command prompt:

      npm install -g yo generator-code

    2. Jei reikia, setting'uose per "editor.tokenColorCustomizations" ir "editor.tokenColorCustomizations" galima patiūninti ir tada sukurti *.color-theme.json:

      Customize your colors in your user settings then generate a theme definition file with the Developer: Generate Color Theme From Current Settings command.

    3. Naujame kataloge padedame sugeneruotą *.color-theme.json ir tame kataloge įvykdome:

      > yo code

      Keliaujame per vedlį:

      • ...
      • rezultate sukuriamas Extension'o katalogas

    4. Sugeneruotą Extension'o katalogą padedame į C:\Users\zm\.vscode\extensions:

      To load an extension, you need to copy the files to your VS Code extensions folder .vscode/extensions. Depending on your platform, it is located in the following folders:

      • Windows %USERPROFILE%\.vscode\extensions
      • macOS ~/.vscode/extensions
      • Linux ~/.vscode/extensions
    5. Perkrauname VS Code'ą, naujas Color Theme'as jau yra.

    Color theme'o override'inimas setting'uose (customize color theme)

    Visual Studio Customizing a Color Theme

    Setting'ų faile: 

    "workbench.colorCustomizations": {
  "editor.background": "#1d1f23"
}

     

     

    • tab.activeModifiedBorder: Border on the top of modified (dirty) active tabs in an active group.
    • tab.inactiveModifiedBorder: Border on the top of modified (dirty) inactive tabs in an active group.
    • tab.unfocusedActiveModifiedBorder: Border on the top of modified (dirty) active tabs in an unfocused group.
    • tab.unfocusedInactiveModifiedBorder: Border on the top of modified (dirty) inactive tabs in an unfocused group.

     

     

     

    Redaktoriaus kodo programavimo kalbos spalvinimas (syntax highlighting colors)

    Visual Studio Code syntax highlighting colors

    TextMate theme

    TextMate grammars consist of a set of regular expression that are used to create a syntax tree out of the source code. 

     

    Each tree node spans a source range and represents a scope. Scopes have a name and stand for either code sections (such as functions, blocks, comments) or symbols (for example keywords, numbers, operators).

     

     

    Override'inimas setting'uose (to tune the editor's syntax highlighting colors)

    [C][Developer: Inspect TM Scopes] — kodo struktūrinio formatavimo tyrinėjimui

    Setting'ų faile taip override'ina visų programavimo kalbų spalvinimą visuose VS Code'o color themes'uose:

    "editor.tokenColorCustomizations": {
  "comments": "#6C768E"
}

    Čia galima spalvinti nedaug:

    A pre-configured set of syntax tokens ('comments', 'strings', ...) is available for the most common constructs.

    Norint daugiau:

    If you want more, you can do so by directly specifying TextMate theme color rules

    "editor.tokenColorCustomizations": {
  "textMateRules": [
    ...
  ]
}

    Override'inti tik konkretų Color theme'ą

    Again, to customize a specific theme only, use the following syntax:

    "editor.tokenColorCustomizations": {
  "[Monokai]": {
    "comments": "#229977"
  }
}
