    Margin'sų collapsing'as

    Kurie margin'sai collapse'uoja

    CSS1 specifikacijoje nustatyta, kad vertikalūs margin'sai collapse'inasi:

    The CSS1 specification, as it defined margins, also defined that vertical margins collapse.

    Naujesnėse specifikacijose nustatyta lanksčiau, kad margin'sai collapse'uoja block direction kryptimi

    Margins only collapse in the block direction, such as between paragraphs.

    O kokia tai kryptis, vertikali ar horizontali, priklauso nuo writing mode:

    In CSS2, only vertical margins are specified to collapse — that is the top and bottom margins on an element if you are in a horizontal writing mode.

    Kaip vyksta collapsing'as

    When margins collapse, they will combine so that the space between the two elements becomes the larger of the two margins. The smaller margin essentially ending up inside the larger one.

    Kodėl buvo įvestas margin'sų collapsing'as

    Margin collapsing makes sense if you consider that in those early days, CSS was being used as a documenting formatting language. Margin collapsing means that when a heading with a bottom margin, is followed by a paragraph with a top margin, you do not get a huge gap between those items.

     

    Kokiomis sąlygomis vertikalūs margin'sai collapse'ina

    Margins collapse in the following situations:

    • Adjacent siblings
    • Completely empty boxes
    • Parent and first or last child element

    A new Block Formatting Context (BFC) will also prevent margin collapsing through the containing element. If we look again at the example of the first and last child, ending up with their margins outside of the wrapper, and give the wrapper display: flow-root, thus creating a new BFC, the margins stay inside.

    Yra CSS savybės, su kuriomis tagui margin'sų collapsing'as neveikia

    Kokios CSS savybės išjungia margin'sų collapsing'ą:

    Margins never collapse if an item has absolute positioning, or is floated.

    Margin'sų collapsing'o tarp gretimų tagų neveikia Flex ir Grid tagų vaikams:

    Flex and Grid containers establish Flex and Grid formatting contexts for their children, so they have different behavior to block layout. One of those differences is that margins do not collapse
