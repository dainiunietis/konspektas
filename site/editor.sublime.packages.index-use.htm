    RegReplace

    https://github.com/facelessuser/RegReplace

    http://facelessuser.github.io/RegReplace/ — vadovas-aprašymas

    Kokia RegExp sintaksė naudojama-veikia šiame plugin'e:

    Remember that the regular expression engine that is used is Python’s re, not Sublime’s internal regular expression engine. If enabling extended_back_references, additional syntax is added which is covered in the backrefs documentation.

    To enable such features as case insensitivity or dotall, see re’s documentation.

    Yra dviejų tipų išsaugomi find-replace'ai (two types of rules that can be created):

    • scope rules (arba regex rules) (with optional scope qualifiers) (Regex with optional scope qualifiers)
    • scope searches (arba scope regex) that apply regex to the targeted scopes (Scope search with regex applied to scope region)

    Find-replace'o tipas: scope rules

    Galimi find-replace'o rule'so parametrai:

     - find (required)
 - replace
 - literal
 - literal_ignorecase
 - greedy
 - scope_filter
 - plugin
 - args

    Minimalus find-replace'as:

    "html5_remove_deprecated_type_attr": {
    "find": "",
    "replace": "",
    "greedy": true
}

     

    Find-replace'o tipas: scope searches

     

    http://facelessuser.github.io/RegReplace/usage/#create-find-and-replace-sequences

    Depending on how the test command was configured, it may cause the panel to close, or you might accidentally close it by pressing esc or running some other command. When closed, the currently opened rule is not lost and can be brought back by running the command palette command RegReplace: Show Edit Panel (the command will only work if the panel has been opened at least once).

    SideBarEnhancements

    (~PS) [files, side bar]

    https://github.com/titoBouzout/SideBarEnhancements

    https://github.com/titoBouzout/SideBarEnhancements/tree/st3

    Šio package'o komandas galima vykdyti per:

    Side bar'o kontekstinį meniu

    jis apibrėžtas: ~PS\Side Bar.sublime-menu

    Pasirinkta komanda vykdoma su Side bar'e (projekto) pažymėtais failais.

    [Tools > Command Palette...]

    jos apibrėžtos: ~PS\Commands.sublime-commands

    Pasirinkta komanda vykdoma su aktyviu-redaguojamu failu.

    ir žinoma priskirtais keybindings'ais

     

    Window'sinės operacijos, t. y. rezultate paliekame ST

     

    Operacijos su failais

    CP: File: New File Relative to Current View

    SBKM: [New File...]

    Py: "command": "side_bar_new_file"

    Sukuria naują failą kataloge, kuriame iškviestas kontekstinis meniu Side bar'e.

    alt+s

    CP: File: Rename

    Py: "command": "side_bar_rename"

    Pervardina redaguojamą-aktyvų failą.

    alt+shift+s

    CP: File: Duplicate

    Py: "command": "side_bar_duplicate"

    CP: File: Move

    SBKM: [Move...]

    Py: "command": "side_bar_move"

    Perkelia failą į kitą katalogą.

    ---

    CP: File: Delete

    SBKM: [Delete]

    Py: "command": "side_bar_delete"

    Perkelia į Recycle Bin'ą

    Teksto failuose paieška

    SBKM: [Find & Replace…]

    Atidaro [Find > Find in Files...] ir į laukelį "Where" įveda ant ko iškviestas kontekstinis meniu Side bar'e, pasirinktuose failuose ar pasirinkto katalogo failuose

    SBKM: [Find In Project…]

    Atidaro [Find > Find in Files...] ir į laukelį "Where" įveda <project> — paieška bus vykdoma visuose projekto kataloguose.

    SBKM: [Find Advanced > In Parent Folder…]

    Atidaro [Find > Find in Files...] ir į laukelį "Where" įveda pasirinkto failo ar katalogo katalogą.

    SBKM: [Find Advanced > In Project Folder…]

    Atidaro [Find > Find in Files...] ir į laukelį "Where" įveda pasirinkto failo/ų ar katalogo/ų "šakninį" projekto katalogą, pvz.: turime projektą apibrėžiantį failą *.sublime-project

    {
  "folders":
  [
    { "path": "/C/kažkas-1" }, < šakninis projekto katalogas
    { "path": "/C/kažkas-2" }, < kitas šakninis projekto kat.
  ]
}

    kokiame projekto šakniniame kataloge kas nors pasirinkta, tą ar tuos šakninius projekto katalogus ir įves į "Where" laukelį.

    SBKM: [Find Advanced > In Parent Folders…]

    Atrodo daro tą patį kaip ir SBKM: [Find In Project…]

    SBKM: [Find Advanced > In Files With Extension(s) XXX]

    Atidaro [Find > Find in Files...] ir į laukelį "Where" įveda pasirinkto failo ar failų tipų filtrus, pvz.: jei Side bar'e kontekstinis meniu buvo iškviestas ant index.htm, tada — Where: *.htm.

    Failų paieška

    CP: File: Search Files — ieško nuo aktyvaus-redaguojamo failo katalogo.

    SBKM: [Find Advanced > In Paths Containing…] — ieško nuo Side bar'e pasirinkto katalogo.

    Py: "command": "side_bar_find_files_path_containing"

    Pačiame redaktoriaus lange-buferyje filtruoja failus su visu keliu nuo pasirinkto katalogo, apima ir visus vaikinius katalogus. Rastų failų atidaryti negalima, galima nusikopijuoti patį kelią.

    Kopijuojame failų kelius

    alt+c,alt+shift+c

    SBKM: [Copy as Text > Name Encoded]

    Py: "command": "side_bar_copy_name_encoded"

    Gauname: index%20---%20GABALELIAI.php

    alt+c,alt+shift+p

    SBKM: [Copy as Text > Path as URI]

    Py: "command": "side_bar_copy_path_encoded"

    Gauname: file:///D:/apache_sites/sites/1974films/index%20---%20GABALELIAI.php

    alt+c,alt+a

    SBKM: [Copy as Text > Absolute Path From Project]

    Py: "command": "side_bar_copy_path_absolute_from_project"

    Gauname: /index --- GABALELIAI.php

    alt+c,alt+shift+a

    SBKM: [Copy as Text > Absolute Path From Project Encoded]

    Py: "command": "side_bar_copy_path_absolute_from_project_encoded"

    Gauname: /index%20---%20GABALELIAI.php

    Operacijos su Projektu

    SBKM: [Copy as Text > Project Folders]

    Py: "command": "side_bar_copy_project_directories"

    Kopijuoja visus projekto šakninius katalogus. Veikia nepriklausomai ant kokio katalogo ar failo buvo iškviestas kontekstinis meniu.
