    Tagas HTML

    Patartina, kad veiktų vartotojo nustatymai naršyklėje:

    leisti paveldėti tokį patį naršyklės bazinį šrifto dydį

    nieko nenustatyti arba tą patį darytų — html { font-size: 100% | 1em }

    arba scale'inti naršyklės bazinį šrifto dydį su relative lengths

    html { font-size: 1.2em }

     

    Jeigu <html> ar <body> nustatytas absolute length, tada naršyklės bazinis šrifto dydis visai neturi įtakos puslapio teksto dydžiams, pvz.:

    html { font-size: 20px }

    Išimtis būtų jei:

    html { font-size: 1em }   // ar 1.2em
body { font-size: 20px }

    Tada dar būtų galima su rem matavimo vienetais nustatyti teksto, ir bet ko kito, dydžius, kurių realus dydis pikseliais keistųsi, pakeitus naršyklės bazinį šrifto dydį, pvz.: 

    p { font-size: 1rem }

     

    Naršyklės bazinis šrifto dydis / browser's font size settings

    Naršyklės turi bazinio šrifto dydžio nustatymą, kurį vartotojas gali keisti. Tas dydis išreiškiamas px.

    Tą dydį paveldi <html> tagas, todėl html { font-size: 100% | 1em } priims, ir <body> tagui perduos naršyklės bazinį šrifto dydį:

    font-size: 100% listens to the browser's font size settings instead of overwriting it. By default in most browsers, this would place 1em at 16px.

    html { font-size: 1.2em } — scale'ina naršyklės bazinį šrifto dydį.

    Ar font-size: geriau nustatyti su em/rem, ar px?

    Is it better to use ems/rems than px for font-size? | CSS-Tricks

    em/rem naudojimo privalumai:

    • Naršyklės turi bazinio šrifto dydžio nustatymą, kurį vartotojas gali keisti, faktiškai tai yra <html> tago font-size:, tik paveldimi em/rem šį nustatymą daro veiksmingu.

      setting type in px prevents browser settings from making font size adjustments (which some people definitely use)

    • Nors iš principo browser zoom'as vienodai zoom'ina ir em/rem:, ir px:

      But browser zoom is the default method for making everything bigger (including text) these days and it works great even if you use px.

      Bet:

      setting type in relative units maintains greater design fidelity as users use browser zoom (which a lot of people definitely use).

    Kodinimo technikos

    didėjant naršyklės langui, visi paveldimi teksto dydžiai truputį didėja 

    body {
  font-size: calc(1em + 1vw);
}

    Svarbu:

    However, the units used, the order and the operation (px + vw) ARE REQUIRED for a responsive behavior.
