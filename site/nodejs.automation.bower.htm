    Kam skirtas ir ką daro Bower'is

    Bower'is yra node.js'o plugin'as, tiksliau npm (Node’s package manager) package'as.

    Ką daro ir ko nedaro:

    Bower can manage components that contain HTML, CSS, JavaScript, fonts or even image files. Bower doesn’t concatenate or minify code or do anything else - it just installs the right versions of the packages you need and their dependencies.

    Įpatybės:

    Bower is optimized for the front-end.

    Bower'io komandinės eilutės struktūra

    Struktūra:

    >bower <command> [<args>] [<options>]

    help'as per CMD:

    >bower help — išveda sąrašą Bower'io komandų (<command>) ir opcijų (<options>)

    >bower help <command> — išsami info apie komandą

    Bower'io komandos:

    https://bower.io/docs/api/#commands

    Globalus Bower'io, Node'o package'o, instaliavimas

    Reikia teisingai-tinkamai suinstaliuoti  Git for Windows:

    https://github.com/bower/bower#windows-users

    Windows users

    To use Bower on Windows, you must install Git for Windows correctly. Be sure to check the options shown below:

      

    Note that if you use TortoiseGit and if Bower keeps asking for your SSH password, you should add the following environment variable: GIT_SSH - C:\Program Files\TortoiseGit\bin\TortoisePlink.exe. Adjust the TortoisePlink path if needed.

    Darome tai su npm (Node’s package manager):

    $ npm install -g bower

    Bower'io versiją sužinome:

    > bower -v

    Čia https://bower.io/ pažiūrime kokia naujausia versija, jeigu pas save senesnė, atnaujiname:

    npm update -g bower

    Bower'io projekte-tinklalapyje instaliavimas (Setup Bower for a Project)

    Visas Bower'io command-prompt'o / terminal'o komandas vykdome iš šakninio projekto-tinklalapio katalogo (project’s root folder).

    Bower'is instaliuojamas sukuriant failą bower.json šakniniame projekto-tinklalapio kataloge, tai galima padaryti 2 būdais:

    Rankiniu

    The project needs a file called bower.json. Create one in the project’s folder, the contents of the file should be something like:

    {
  "name": "my-project"
}
    Su terminal'o komanda

    > bower init

    Ši komanda paleidžia wizad'ą, kuriame surašius ko prašo, sukurs bower.json failą šakniniame projekto-tinklalapio kataloge.

    Bower'io konfigūravimas

    https://bower.io/docs/config/

    Bower keeps track of these packages in a manifest file, bower.json.

    Bower'io package'ų instaliavimo vietos-katalogo nustatymas

    By default Bower is going to download everything to the bower_components folder. We can change this using the directory setting in the .bowerrc file. If this file doesn’t yet exist you will need to add it to the same folder that contains your bower.json file. This file is a JSON file for defining your Bower settings. To change the folder that Bower installs packages add the directory setting like:-

    {    "directory": "public/vendor"  }

    This will cause Bower to download everything to public/vendor/ instead of bower_components.

    Bower'io package'ų radimas

    https://bower.io/search/

    Finding Packages

    There are a few of ways of finding Bower packages. You can search via the Bower website or alternatively you can use the command line.

    To search for packages from the terminal you use Bower’s search command followed by what you’re searching for:-

    $ bower search <query>

    For example, to search for jQuery packages:-

    $ bower search jquery

    Try it yourself. You should see a very long list of available packages.

    Bower'io package'ų valdymas: instaliavimas, išinstaliavimas, atnaujinimas

    Bower'is rūpinasi dependents'ais:

    Another neat trick that Bower will perform is that if you attempt to install a package that depends on another one it will download both and make sure the packages are compatible.

    Konkretaus package'o instaliavimas

    Instaliuojama su terminal'o komanda:

    > bower install <package> --save

    Ši komanda:

    • Sukuria (kai iškviečiama pirmą kartą projekte) katalogą bower_components šakniniame projekto-tinklalapio kataloge.
    • Iš interneto atsiunčia nurodytą <package>, ir padeda į bower_components katalogą.

    • The --save flag tells Bower to add the package to the bower.json file for future reference.

      install a package and add it to bower.json

    Konkrečios Bower'io package'o versijos instaliavimas (install specific version of a package and add it to bower.json):

    > bower install <package>#<version> --save
konkrečiai:
> bower install jquery#1.8 --save

    Konkretaus package'o išinstaliavimas-pašalinimas

    bower uninstall <package-name> --save

    Ką daro flag'as --save:

    The --save flag will remove the package from the bower.json file as well as uninstall it.

    Bower'io package'ų instaliavimas pagal bower.json (install dependencies listed in bower.json)

    Čia matyt tam atvejui, kai turi kokį tai bower.json , kuriame yra įrašai apie suinstaliuotus package'us, ir jį padedi į kito-naujo-tuščio projekto katalogą.

    > bower install

    Visų Bower'io package'ų, t. y. pagal projekto bower.json, atnaujinimas

    > bower update
