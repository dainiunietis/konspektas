    Textual data types

    <ident>

    ident - CSS: Cascading Style Sheets | MDN

    CSS identifier — CSS Values and Units Module Level 3

    Tai yra bazinis duomenų tipas kitiems tipams išvesti:

    As this type is not a real type but a convenience type used to simplify the definition of other CSS syntax.

    Reikalavimai pačiam <ident>:

    Gali būti naudojami nurodyti simboliai

    It consists of one or more characters, where characters can be any of the following:

    • any alphabetical character (A to Z, or a to z),
    • any decimal digit (0 to 9),
    • a hyphen (-),
    • an underscore (_),
    • an escaped character (preceded by a backslash, \),
    • a Unicode character (in the format of a backslash, \, followed by one to six hexadecimal digits, representing its Unicode code point)
    Apribojimai pirmam simboliui

    Moreover, the first character must not be a decimal digit, nor a hyphen (-) followed by a decimal digit or another hyphen.

    Negali būti apskliaustas:

    A <custom-ident> must not be placed between single or double quotes as this would be identical to a <string>.

    Savybės:

    is case-sensitive

    Note that id1, Id1, iD1 and ID1 are all different identifiers as they are case-sensitive.

    Pvz.:

    Valid identifiers

    nono79      	A mix of alphanumeric characters and numbers
ground-level	A mix of alphanumeric characters and a dash
-test       	A dash followed by alphanumeric characters
--toto      	A custom-property like identifier
_internal   	An underscore followed by alphanumeric characters
\22 toto    	A Unicode character followed by a sequence of alphanumeric characters
bili\.bob   	A correctly escaped period

    Invalid identifiers

    34rem   	It must not start with a decimal digit.
-12rad  	It must not start with a dash followed by a decimal digit.
bili.bob	Only alphanumeric characters, _, and - needn't be escaped.
'bilibob'	This would be a <string>.
"bilibob"	This would be a <string>.

    <custom-ident>

    <custom-ident> - CSS: Cascading Style Sheets | MDN

    Author-defined Identifiers: the <custom-ident> type — CSS Values and Units Module Level 3

    Išvestinis duomenų tipas:

    <custom-ident> – A restricted subset of <ident> values excluding CSS-wide keywords, custom properties and other per-property values.

     

    Pvz.:

    Valid identifiers

    nono79            A mix of alphanumeric characters and numbers
ground-level      A mix of alphanumeric characters and a dash
-test             A dash followed by alphanumeric characters
_internal         An underscore followed by alphanumeric characters
\22 toto          A Unicode character followed by a sequence of alphanumeric characters
bili\.bob         A correctly escaped period

    Invalid identifiers

    34rem             It must not start with a decimal digit.
-12rad            It must not start with a dash followed by a decimal digit.
bili.bob          Only alphanumeric characters, _, and - needn't be escaped.
--toto            It must not start with two dashes. This would be a custom property.
'bilibob'         This would be a <string>.
"bilibob"         This would be a <string>.

     

    <string>

    <string> - CSS: Cascading Style Sheets | MDN

    Tai yra:

    The <string> data type is composed of any number of Unicode characters surrounded by either double (") or single (') quotes.

    Pagal tas pačias CSS taisykles galima string'e užkoduoti Unicode simbolius šešioliktainiais skaičiais:

    All characters can also be represented with their respective Unicode code points in hexadecimal, in which case they are preceded by a backslash (\). For example, \22 represents a double quote, \27 a single quote ('), and \A9 the copyright symbol (©).

    Generiniai-bendri duomenų tipai (generic data types)

     

    All the Generic CSS Data Types

    Generic data types are not related to any specific property. For example we can have a value of 10px, which is a <length> data type, for the margin, font-size or a number of other propertes.

     

    CSS specifikacijoje CSS Values and Units Module Level 3 apibrėžta: keywords <ident> ir visi basic data types <length> <percentage> ..:

    1. keyword values (such as auto, disc, etc.), which appear literally, without quotes (e.g. auto)
    2. basic data types, which appear between < and > (e.g., <length>, <percentage>, etc.).

     

    CSS basic data types - CSS: Cascading Style Sheets | MDN

    CSS basic data types define typical values (including keywords and units) accepted by CSS properties and functions. They are a special kind of component value type.

     

    Specifiniai duomenų tipai (specific data types)

     

    Specific data types are related a single property or a small group of properties. For example, the <transform-function> data type is used as the value for the transform: property alone.

     

    Textual Data Types (value definition type category-group)

    Keyword'ai (pre-defined keywords)

    Pre-defined Keywords

    value definition type: <ident>

    The <ident> data type refers to the pre-defined keywords in CSS. This includes both the unique values for certain properties, for example block for the display property, as well as the CSS-wide values initial, inherit and unset.

    Pvz.:

    value definition — border-collapse: collapse | separate
    CSS — table { border-collapse: separate }

    CSS — div { color: red }

     

     

    Visų CSS savybių keyword'ai (CSS-wide keywords, global values)

    CSS-wide keywords

    Initial, Inherit, Unset, and Revert

    value definition type: <ident>

    There are four values that theoretically can be applied to any CSS property (besides none). These are the explicit defaulting values - initial, inherit, unset, and revert. 

     

    • Use unset to set a property to its inherited value if it inherits, or to its initial value if not.
    • Use revert to reset a property to the value established by the user-agent stylesheet (or by user styles, if any exist).
    • Use inherit to make an element's property the same as its parent.
    • The all property lets you reset all properties to their initial, inherited, reverted, or unset state at once.

     

    Keyword'as — initial

    initial - CSS: Cascading Style Sheets | MDN

    CSS — data types — CSS-wide keywords

     

    CSS savybei priskiria Pradinė-default'inė reikšmė (initial value):

    The initial CSS keyword applies the initial (or default) value of a property to an element. It can be applied to any CSS property. 

     

    Note: On inherited properties, the initial value may be unexpected. You should consider using the inherit, unset, or revert keywords instead.

    Keyword'as — inherit

    inherit - CSS: Cascading Style Sheets | MDN

    The inherit CSS keyword causes the element for which it is specified to take the computed value of the property from its parent element.

    kaip veikia inherit keyword'as paveldimoms ir nepaveldimoms CSS savybėms

    ...

    be child class'o — paveldi paveldimas CSS savybes iš parent'o
    su child class'u — paveldimoms ir nepaveldimoms CSS savybėms nustatytos kitos reikšmės
    su child class'u ir tage nustatytu inherit joms — paveldi visas CSS savybes iš parent'o

    Keyword'as — unset

    unset - CSS: Cascading Style Sheets | MDN

    Paveldimoms CSS savybėms unset priskiria inherit, o nepaveldimoms — initial:

    The unset CSS keyword resets a property to its inherited value if it inherits from its parent, and to its initial value if not.

    When unset is applied to a property, it will apply either initial or inherit, depending on what the property's default behaviour is. If the property by default inherits, the inherit will be applied. Otherwise, initial will be applied.
