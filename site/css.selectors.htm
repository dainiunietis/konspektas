    Selektorių grupavimas / Grouping selectors, Selector list

    Selector list - CSS: Cascading Style Sheets | MDN

    Kableliu (comma) atskirti selektoriai savo suselektintiems tagams priskiria tas pačias deklaracijas, tai vadinama selektorių grupavimu. Taigi grupavimas leidžia skirtingiems selektoriams priskirti tą patį deklaracijų bloką.

    // Single Line Grouping
p, pre { ... }

// Multi Line Grouping
p,
pre { ... }

     

     

    Selector list invalidation

    Vienas neteisingas selektorius "išjungia" visą selektorių grupę.

    A downside to using selector lists is that the following aren't equivalent:

    h1 { ... }
h2:maybe-unsupported { ... }
h3 { ... }

h1, h2:maybe-unsupported, h3 { ... }

    This is because a single unsupported selector in a selector list invalidates the whole rule.

    Yra sprendimas su :is():

    A way to remedy this us to use the :is() or :where() selectors, which accept a forgiving selector list. This will ignore invalid selectors in the list but accept those which are valid.

    :is(h1, h2:maybe-unsupported, h3) { ... }

    Flag'as case-insensitive

    Default'iškai visos string'ų palyginimo operacijos yra case-sensitive:

    input[type="email"] {}

    <input type="email">
<input type="Email">

    Kad būtų case-insensitive: 

    input[type="email" i] {}

    <input type="email">
<input type="Email">

    Operatorius ~=

    Tago atributo reikšmė padalinama joje esančiais tarpais (a whitespace-separated list of words) (tiksliau gali būti bet kokie withespace'o simboliai) į masyvą, atributas laikomas atitinkančiu selektoriaus sąlygą, jei bent vienas to masyvo elementas lygus nurodytam selektoriuje. 

    [class~="tas"] {}
[class~="suris"] {}

    Location pseudo-classes — links

    :link

    :link - CSS: Cascading Style Sheets | MDN

    Selektina neaplankytus tagus <a> <area>. Tai reiškia turi atributą href, kuriame nurodytas adresas nebuvo aplankytas, tksliau nėra naršyklės aplankytų puslapių-adresų istorijoje.

    The :link CSS pseudo-class represents an element that has not yet been visited. It matches every unvisited <a>, <area>, or <link> element that has an href attribute.

    Refers to any anchor that is a hyperlink (i.e., has an href attribute) and points to an address that has not been visited.

     

    :visited

    :visited - CSS: Cascading Style Sheets | MDN

    Selektina aplankytus tagus <a> <area>.

    The :visited CSS pseudo-class represents links that the user has already visited.

    Apie "privacy reasons"

    Privacy and the :visited selector - CSS: Cascading Style Sheets | MDN

    Privacy restrictions

    For privacy reasons, browsers strictly limit which styles you can apply using this pseudo-class, and how they can be used:

    • Allowable CSS properties are color, background-color, border-color, border-bottom-color, border-left-color, border-right-color, border-top-color, column-rule-color, outline-color, text-decoration-color, and text-emphasis-color.
    • Allowable SVG attributes are fill and stroke.
    • The alpha component of the allowed styles will be ignored. The alpha component of the element's non-:visited state will be used instead, except when that component is 0, in which case the style set in :visited will be ignored entirely.
    • Although these styles can be change the appearance of colors to the end user, the window.getComputedStyle method will lie and always return the value of the non-:visited color.
    • The <link> element is never matched by :visited.

    :any-link

    :any-link - CSS: Cascading Style Sheets | MDN

    Selektina ir neaplankytus, ir aplankytus tagus <a> <area>.

    The :any-link CSS pseudo-class selector represents an element that acts as the source anchor of a hyperlink, independent of whether it has been visited. In other words, it matches every <a>, <area>, or <link> element that has an href attribute. Thus, it matches all elements that match :link or :visited.

     

     

     

     

     

    :local-link

    :local-link - CSS: Cascading Style Sheets | MDN

    The :local-link CSS pseudo-class represents an link to the same document. Therefore an element that is the source anchor of a hyperlink whose target's absolute URL matches the element's own document URL.

    User action pseudo-classes

    :hover

     

    The :hover CSS pseudo-class matches when the user interacts with an element with a pointing device, but does not necessarily activate it. It is generally triggered when the user hovers over an element with the cursor (mouse pointer).

    Veikimas planšetėse-touchscreens'uose ribotas ir prastai standartizuotas:

    Note: The :hover pseudo-class is problematic on touchscreens. Depending on the browser, the :hover pseudo-class might never match, match only for a moment after touching an element, or continue to match even after the user has stopped touching and until the user touches another element. Web developers should make sure that content is accessible on devices with limited or non-existent hovering capabilities.

     

    :active

    Selektina bet kurį elementą, ant kurio yra paspausta pelės žymekliu (arba paspausta pirštu planšetėse-touchscreens'uose), t. y. esant pelės žymekliui virš elemento, paspaustas bet kuris pelės klavišas: kairysis, dešnysis, ar ratukas. Atleidus pelės klavišą — :active būsena baigiasi.

    Refers to any element that has been activated by user input—e.g., a hyperlink on which a user clicks during the
    time the mouse button is held down.

    The :active CSS pseudo-class represents an element (such as a button) that is being activated by the user. When using a mouse, "activation" typically starts when the user presses down the primary mouse button.

     

    Ištikrųjų paspaustas sufokusuojamas elementas atitinka :focus:active selektorių:

    When clicked the button is in :focus:active state.

     

     

    :focus

    Selektina sufokusuotą elementą, kuris priima klaviatūros įvedimą.

    Refers to any element that currently has the input focus—i.e., can accept keyboard input or be activated in some way.

    :focus represents the state when the element is currently selected to receive input.

    The :focus CSS pseudo-class represents an element (such as a form input) that has received focus. It is generally triggered when the user clicks or taps on an element or selects it with the keyboard's Tab key.

    Vienu metu sufokusuotas gali būti tik vienas tagas-elementas visame dokumente.

    Sufokusuojami elementai, kurie gali priimti klaviatūros įvedimą:

    • <a href="">
    • <button>
    • Formos elementai: <input> ...
    • bet kuris elementas su tabindex="0..N"

    Sufokusuojamų elementų fokusavimo būdai:

    • Su Tab.
    • Su pelės kursorių ant elemento ir paspaudus pagrindinį kairyjį arba kontekstinį dešnįjį klavišą.

    :focus ir :active veikimas

    There are four cases.

    1. By default, active and focus are both off.
    2. When you tab to cycle through focusable elements, they will enter :focus (without active).
    3. When you click on a non-focusable element, it enters :active (without focus).
    4. When you click on a focusable element it enters :active:focus (active and focus simultaneously).

    Accessibility

    Accessibility concerns

    Make sure the visual focus indicator can be seen by people with low vision.  This will also benefit anyone use a screen in a brightly lit space (like outside in the sun). WCAG 2.1 SC 1.4.11 Non-Text Contrast requires that the visual focus indicator be at least 3 to 1.

    :focus { outline: none; } — tokio rule'o neturi būti:

    Never just remove the focus outline (visible focus indicator) without replacing it with a focus outline that will pass WCAG 2.1 SC 1.4.11 Non-Text Contrast.
