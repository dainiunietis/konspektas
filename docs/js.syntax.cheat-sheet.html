    Value vs. Reference Variable Assignment

    nas5w/javascript-tips-and-tidbits: A continuously-evolving compendium of javascript tips based on common areas of confusion or misunderstanding.

    When JavaScript assigns one of the five primitive type (i.e., Boolean, null, undefined, String, and Number) to a variable, the JavaScript runtime gets to determine whether that primitive is assigned by reference or by value. It doesn't really matter how it's done because primitives can't be mutated (they're immutable). However, when the assigned value is an Array, Function, or Object a reference to the array/function/object in memory is assigned.

    Arrow Functions

    Arrow functions, the basics

    Su keliais argumentais:

    // Arrow function
let sum = (a, b) => a + b;

// Function expression
let sum = function(a, b) {
  return a + b;
};

    Kai tėra vienas argumentas, nereikia jo apskliausti:

    let double = n => n * 2;
let double = function(n) { return n * 2 }

    Kai nėra argumentų, reikia tuščių skliaustelių: 

    let sayHi = () => alert("Hello!");
let sayHi = function() { alert("Hello!") }

    Duomenų tipų konvertacija į kitus duomenų tipus / Type Conversions

    Type Conversions

    Pagal kontekstą vykdomas automatinis konvertavimas:

    Most of the time, operators and functions automatically convert the values given to them to the right type.

    For example, alert automatically converts any value to a string to show it. Mathematical operations convert values to numbers.

    Ir galima įvykdyti norimą konvertaciją — explicit convertion:

    There are also cases when we need to explicitly convert a value to the expected type.

    Konvertacija į string'us / string conversion

    Automatinis:

    alert(false); // 'false'

    Nurodomas: 

    str = String(true); // str == 'true'
str = String(47);   // str == '47'

    Konvertacija į skaičius / numeric conversion

    Automatinis:

    alert( "6" / "2" ); // 3

    Nurodomas: 

    //--- strings ----------------------------

num = Number('123');   // num == 123
num = Number('');      // num == 0

// Whitespaces from the start and end are removed
num = Number(' 123 '); // num == 123
num = Number('  ');    // num == 0

// conversion failed
num = Number('aaa'); // num == NaN

//--- boolean ----------------------------

num = Number(true);    // num == 1
num = Number(false);   // num == 0

//--- ... ----------------------------

num = Number(undefined); // num == NaN
num = Number(null);      // num == 0

    Konvertacija į loginę reikšmę / boolean conversion

    Automatiškai vykdoma in logical operations., pvz.:

    if (5) {
  ...
}

    The conversion rule:

    • Values that are intuitively "empty", like 0, an empty string, null, undefined, and NaN, become false.
    • Other values become true.
    //--- skaičiai ----------------------------

bln = Boolean(1);           // bln == true
bln = Boolean(-5);          // bln == true

bln = Boolean(0);           // bln == false

//--- strings ----------------------------

bln = Boolean('belekas');   // bln == true
bln = Boolean('  ');        // bln == true
bln = Boolean('0');         // bln == true

bln = Boolean('');          // bln == false

//--- ... ----------------------------

bln = Boolean(null);       // bln == false
bln = Boolean(undefined);  // bln == false
bln = Boolean(NaN);        // bln == false

    Array.filter() — masyvo filtravimas su predicate funkcija

    Kiekvienas masyvo elemnetas patikrinamas metodui .filter perduota funkcija — jei įvykdžius su jai perduotu masyvo elementu grąžina true — tas elementas lieka grąžinamame masyve, pvz.:

    const input = [1, 2, 3]
const predicate = ArrElm => ArrElm > 1
const output = input.filter(predicate) // [2, 3]
