    WebStorm'o projektai

    Working with WebStorm projects - Help | WebStorm

    Project - Help | WebStorm

    Kas apskritai yra WS projektas:

    Whatever you do in WebStorm, you do that in the context of a project. A project is an organizational unit that represents a complete software solution. It serves as a basis for coding assistance, bulk refactoring, coding style consistency, etc.

    Kas konkrečiai:

    A project in WebStorm is a folder with the source code your edit, the libraries and tools you use (for example, in the node_modules subfolder), and various app configuration files (for example, package.json or .eslintrc).

    Project / Project files - Help | WebStorm

    project directory — projekto šakninis-pagrindinis katalogas, apibrėžiamas vieninteliu dalyku — subkatalogu .idea/, kuriame saugomi projekto nustatymai. WS projekte gali būti tik vienas project directory.

    project directory automatiškai yra ir content root:

    A project has at least one content root folder which by default is the project folder itself.

    Content Root - Help | WebStorm

    content root'o subkatalogų tipai

    Configuring Project Structure / Configuring folders in a content root - Help | WebStorm

    Svarbu suprasti, kad tipai nustatomi konkretiems katalogams, t. y. katalogų medyje konkrečioje šakoje, ir konkretaus pavadinimo.

    Kad pakeisti projekto failų "dalyvavimą" IDE veikloje, katalogams galima priskirti kitus tipus:

    By default, all the files in a content root folder are involved in indexing, searching, parsing, code completion, etc. To change this status, folders within a content root can be assigned to the following categories:

    • Test source roots
    • Resource roots
    • Excluded roots

    Projekto struktūrą, t. y. subkatalogų tipus, galima nustatyti:

    • [S][Directories]
    Source

    Pagal nutylėjimą visi content root'o subkatalogai, įskaitant ir patį content root'ą — yra Source tipo, tai reiškia, kai nenustatytas joks kitas tipas:

    By default, any folder is treated as Source.

    Tai reiškia:

    all the files in a content root folder are involved in indexing, searching, parsing, code completion, etc

    Test Source Root

    Tests folders let you keep code related to testing separately from production code.

    Resource Root

    Files in Resource Root folders can be referenced relative to these folders instead of specifying full paths.

    Jei teisingai suprantu, vienintelis savybė, kurią suteikia Resource Root'as, yra tas pats kaip sąlyginis failų adresavimas tinklapyje, t. y. Resource Root'as yra tinklapio šakninis katalogas, pvz.:

    Excluded Root

    Excluded roots (shown as ) are ones that WebStorm "almost ignores". These roots contain files and folders ignored by WebStorm when indexing, searching, parsing, watching etc.

    Excluded roots are not visible to WebStorm. Usually, one would like to exclude temporary build folders, generated output, logs, and other project output. Excluding the unnecessary paths is a good way to significantly improve performance.

    Project Tool Window'se galima rodyti arba paslėpti Excluded Root katalogus

    Galima exclude'inti konkretų failą Project Tool Window'se — Configuring Project Structure / Excluding files - Help | WebStorm

    Scope

    Scope - Help | WebStorm

    Kas tai yra ir ką daro:

    A scope() is a subset of files, and/or directories in your project, to which you can limit the application of specific operations, e.g. search, code inspection, etc. Besides, you can configure coloring for each scope to see at once what sort of file you are dealing with.

    Kokie yra scope'ai:

    • a number of predefined scopes that cover basic cases
    • it is possible to add custom scopes to your project

    Serveryje esančio projekto redagavimas arba, kitaip sakant, lokalaus projekto sinchronizavimas su serveryje esančiu

    WebStorm does not support direct editing of files on remote hosts. Therefore to work with remote sources in WebStorm, you need to download them, open the folder where they are stored, and arrange them in a WebStorm project, as described in Creating a project from existing local sources. To keep your local and remote sources synchronized, configure automatic upload using the Upload changed files automatically to the default server drop-down list in the Options dialog box.
