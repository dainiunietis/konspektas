    http://www.sublimetext.com/docs/2/tab_completion.html

    Ši komanda (Tab Completion'as ar Tab auto-completion'as) ieško prieš kursorių esančiam žodžiui tinkamiausio completion ar snippet'o, jeigu randa — padaro auto-completion'są, t. y. expand'ina rastąjį, jeigu ne — įveda tab'o simbolį, completions meniuso neišveda.

    Šios komandos veikimas priklauso nuo setting'o tab_completion, jeigu tab_completion = false, tada ši komanda completion'a tik snippet'us, jeigu tab_completion = true, tada ir snippet'us, ir completion'us.

    Dabar man atrodo, kad geriau turėti setting'ą "tab_completion": false, nes tada tabas tik expand'ins snippets'us, taip galima turėti savo snippet'ų sistemą ir būtent su tab'u juos expand'inti.

    Commit on Tab

    By default, the selected item in the completion popup will be committed when enter is pressed. This can create ambiguity between committing the completion, and inserting a newline. By setting the auto_complete_commit_on_tab setting to true, enter will insert a newline, and tab will commit the current completion. There are other benefits, too: because Sublime Text knows there is no ambiguity, it will show a more curated list of completions, with the one you want more likely to be in first place.

    Enabling Commit on Tab is recommended, but it will take a short time to get used to.

    Parodo Completions'ų meniusą:

    Jeigu prieš kursorių nėra žodžio

    Parodo Completions'ų meniusą su visais elementais.

    Jeigu prieš kursorių yra žodis

    Parodo atfiltruotą Completions'ų meniusą — įtraukiami tik tie elementai, kurie prasideda tuo žodžiu.

    Jeigu yra vienintelis galimas Completions'ų meniuso elementas — Completions'ų meniusas neišvedamas, o iš karto expand'inamas tas vienintelis elementas.

     

    The completions list (Ctrl+spacebar) does its work in two ways: by bringing up a list of suggested words to be completed, or by inserting the best match directly.

     

    Kai jau parodytas Completions'ų meniusas — ctrl+space važiuoja per šį meniusą.

    Completions'ų meniusas formuojamas iš šių šaltinių ir pirmesni šaltiniai yra svarbesni, t. y. rodomi pirmiau, jei nėra svarbesnių kriterijų:

    • *.sublime-snippet
    • API-injected completions
    • *.sublime-completions
    • Words in buffer

    Kaip pasirenkama ką automatiškai expand'inti ir kaip formuojamas Completions'ų meniusas:

    Snippets will always win if the current prefix matches their tab trigger exactly. For the rest of the completion sources, a fuzzy match is performed. Furthermore, snippets always lose with fuzzy matches.

    But this is relevant only when the completion is inserted automatically. When a list of completions is shown, snippets will still be listed alongside the other items, even if the prefix only partially matches the snippets’ tab triggers.

    Operacinės user'io kataloge

    ~/.codeintel/config (~ yra C:\Documents and Settings\Administrator\)

    Projekto kataloge

    projekto_katalogas/.codeintel/config

    Pavyzdys yra Packages\SublimeCodeIntel\.codeintel\config.

    Package'o katalogas operacinės user'io kataloge

    Šis package'as sukuria ir naudoja C:\Documents and Settings\Administrator\.codeintel\

    Sprendimas problemoms spręsti:

    Troubleshooting

    To force re-indexation of the code intelligence database you need to follow these steps:

    • Close Sublime Text 2
    • Open a terminal or navigate through your directories to find the directory ~/.codeintel that contains codeintel.log, VERSION and the directory db. In Windows, this should be at%userprofile%\.codeintel.
    • Delete the whole directory ~/.codeintel and all of its content. Particularly, if you want to delete only the indexes, the code intelligence database indexes are located inside ~/.codeintel/db.
    • Start Sublime Text 2 and enjoy a clean re-indexing!
