    History navigation

    Navigating input history is now more natural and consistent across VS Code. You can use the UpArrow (history.showPrevious) and DownArrow (history.showNext) keys to navigate the input boxes in the Search and Problems views and the Find widgets in the Editor, Terminal, and Web views.

    For simplicity and consistency, we changed the default keybindings for navigating among input boxes from UpArrow and DownArrow to Ctrl+UpArrow (or Cmd+UpArrow on macOS) and Ctrl+DownArrow (or Cmd+DownArrow on macOS) keys respectively. You can always go back to previous behavior by updating Ctrl+UpArrow, Ctrl+DownArrow and Ctrl+UpArrow keybindings.

    Note: We also removed all existing 18 History Navigation Commands and unified them into two commands history.showPrevious and history.showNext.

    Automatizacija

     

    You might know that the Sidebar can be toggled with Cmd/Ctrl B. The ActivityBar can also be toggled to make even more room. The Multi-Key Extension allows for multiple commands to be bound to a single keystroke. This keystroke can then hide both the sidebar and the activity bar at the same time.

    Add the following to your User Setings file...

    "multiCommand.commands": [
{
"command": "multiCommand.makeRoom",
"sequence": [
  "workbench.action.toggleSidebarVisibility",
  "workbench.action.toggleActivityBarVisibility"
]
}
]

    Then add a key binding to the keybindings.json file

    {
"key": "alt+b",
"command": "multiCommand.makeRoom"
}

    Paieškos nustatymai "files to include" ir "files to exclude"

    Kokia yra abiejų sintaksė:

    ./

    Žymi projekto šakninį katalogą.

    If you enter ./example, that will match the folder example/ at the top level of your workspace. 

    <žodis>

    Ieškos failuose, kurių kelio dalis-segmentas (a path segment) yra <žodis>:

    • arba pilnas failo pavadinimas su išplėtimu,
    • arba katalogo pavadinimas kelyje (t. y. a path segment), pvz.:
      files to include: sass
      ieško failuose: .\**\sass\**\*
    !<žodis>

    Use ! to exclude those patterns from the search. !example will skip searching any folder or file named example. 

    glob syntax
    • * to match one or more characters in a path segment
    • ? to match on one character in a path segment
    • ** to match any number of path segments, including none
    • {} to group conditions (e.g. {**/*.html,**/*.txt} matches all HTML and text files)
    • [] to declare a range of characters to match (e.g., example.[0-9] to match on example.0, example.1, …)

    Kaip veikia "files to include"

    Kokiuose failuose ieško, kai nieko neįvesta:

    • visuose projekto failuose, kurių
    • atidarytuose ne projekto failuose

    Tipiniai pavyzdžiai:

    files to include: sass

    Ieško failuose:

    • glob syntax:
      **/sass/**/*
    • konkretūs:
      site/assets/sass/kazkas.scss
      site/assets/sass/kitas/kazkas.scss
      sass/kazkas.scss
    -

     

     

    nurodžius absoliutų kelią, tiek pačiame projekte, tiek bet kur kitur diske — "glob syntax" neveikia, veikiantis pvz.:

    files to include: C:\temp
    ieško visuose failuose: C:\temp\*.* ir subkataloguose, t. y. pilnai:
    C:\temp\**\*.*

    ir neveikiantis "glob syntax":

    files to include: C:\te*
    neieško

    Visual Studio Code October 2018

    References View

    There is a new command List All References, which displays a dedicated Side Bar view showing reference search results. This complements the existing Peek view presentation and provides a more stable list of references. In the context menu, select List All References (Shift+Alt+F12) and references are shown in a separate view, just like search results.

    Results are stable and individual results can be cleared, which is great if you use the view as a To Do list. You can use F4 and Shift+F4 to navigate through results without taking your hands off the keyboard.

    Let us know what you think and we'll be refining the experience in our next update.

    Outline View

    ctrl+numpad_multiply(*)

    [CP][File: Focus on Outline]

    [C][outline.focus]

    Outline View'so parodymas-paslėpimas per Sidebar'o Explorer'io title'o kontekstinį meniu:

    It is a separate section in the bottom of the File Explorer. When expanded, it will show the symbol tree of the currently active editor.

    There are several Outline view settings which allow you to enable/disable icons and control the errors and warnings display (all enabled by default):

    • outline.icons - Toggle rendering outline elements with icons.
    • outline.problems.enabled - Show errors and warnings on outline elements.
    • outline.problems.badges - Toggle using badges for errors and warnings.
    • outline.problems.colors - Toggle using colors for errors and warnings.

     

    Grafiniai failai (graphics, images)

    [CP][Emmet: Update Image Size]

    Naudojimas:

    HTML

    <img src="images/kazkas.png">

    prideda:

    <img src="images/kazkas.png" width="753" height="234">

    CSS
    div {
    background: url("../images/kazkas.png");
}

    prideda:

    div {
    background: url("../images/kazkas.png");
    width: 753px;
    height: 234px;
}

    Update Image Size

    VS Code can automatically update HTML img tags with the correct size of images. This works for images that are local to the project as well as images included by URL. It also works in CSS properties like `background-image`.
